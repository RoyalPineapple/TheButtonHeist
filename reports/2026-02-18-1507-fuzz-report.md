## Notable Observations

1. **Back button label is context-sensitive** (expected behavior): When Controls Demo is reached via Touch Canvas's nav stack, the back button reads "Touch Canvas". When reached from root, it reads "ButtonHeist Test App". Correct UIKit navigation stack behavior.

2. **Back navigation triggers DIFF not SCREEN_CHANGED**: Activating Back from Controls Demo or Touch Canvas returns a `DIFF` result (partial update) rather than `SCREEN_CHANGED`. The diff captures only 1–2 element changes during what is a full navigation, likely due to UIKit animation interleaving. The final `get_interface` always shows the correct destination screen. Not a bug — a ButtonHeist observation to be aware of.

3. **Touch Canvas surface is invisible to the accessibility tree**: The drawing canvas in Touch Canvas exposes only Back + Reset buttons. The `UIView` canvas is not in the a11y tree but does accept `draw_path` and other touch gestures. Diamond shape drawn and confirmed via screenshot; cleared successfully via Reset.

4. **Red dot artifact after draw_path**: A small red dot appeared at ~(480, 880) after `draw_path`. It disappeared after Reset. Likely a touch position indicator in the test app, not a bug.

5. **Stepper Decrement disabled at minimum value** (expected behavior): In Adjustable Controls, `buttonheist.adjustable.stepper-Decrement` becomes dimmed (no actions) when the stepper is at 0. Correct pattern.

6. **Swipe actions exposed as named custom accessibility actions** (expected behavior): In Buttons & Actions, `buttonheist.actions.customActionsItem` exposes "Share" and "Favorite" as named custom a11y actions. Correct implementation pattern.
