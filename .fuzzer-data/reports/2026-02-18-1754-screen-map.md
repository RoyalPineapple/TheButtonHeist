## Findings

### F-1 [ANOMALY] Sheet Dismiss button positioned off-screen

**Severity**: Medium
**Screen**: Sheet overlay
**Path**: Alerts & Sheets → Show Sheet
**Element**: `buttonheist.presentation.sheetDismiss`
**Frame**: `{x:171, y:1268, w:59, h:20}` — bottom edge at Y:1288 in an 874pt window
**Behavior**: Activation by identifier fails with "Element not found". Sheet appears to auto-dismiss.
**Root cause hypothesis**: Sheet content is laid out assuming a taller scroll view or the sheet's internal coordinate system is not correctly mapped to screen coordinates. The dismiss button's position at Y:1268 suggests it's placed relative to the sheet's internal content height rather than the visible area.
**Accessibility impact**: The dismiss button is not reachable via accessibility API. VoiceOver users would be unable to dismiss the sheet programmatically.

### INFO: Delta type inconsistency on back-navigation

**Severity**: Low (informational)
**Screens**: Main Menu ↔ Controls Demo Submenu, Main Menu ↔ Touch Canvas
**Behavior**: Back-navigation to Main Menu reports `valuesChanged` or `elementsChanged` delta instead of `screenChanged`. Content is correct — only the delta classification is wrong.
**Impact**: Tools relying on `screenChanged` to detect navigation will miss these transitions.
